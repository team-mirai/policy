## このマニフェスト自身も、みんなの知恵を集めて改善していきます

　チームみらいは全てを決めた後にマニフェストを発表するのではなく、よりオープンにみなさんと一緒にマニフェストを育てていきたいと考えております。具体的には、AI熟議システムの『いどばた』や全国で開催するオフラインミーティング、独自に行う専門家へのインタビューなどを通じて、どんどん知恵を集めたいと思います。

　「考えがまだまとまっていないではないか」とご批判を受けるリスクを積極的に取りながら、未熟なバージョンであっても公開に踏み切ります。なぜそうするかといえば、批判の目をたくさん入れ、意見を取り入れてゆくことが最も良い政策を作るやり方だと思うからです。また、これはシンプルな多数決のように「意見が多いものを取り入れる」ということではありません。意見は聞きますし、プロセスの透明化も行いますが、あくまで意思決定は既存政党と同様にチームみらいの人が行います。これは、責任を取れる『人』が主体となって判断をすることが重要であるという価値観に基づくものです。

　なお今後も議論の結果、これらのマニフェストの内容がまだ変わる可能性がある点はご理解いただけますと幸いです。

## 借地借家法の見直しとAI・データ活用による賃貸市場の健全化

日本の賃貸住宅市場は、長年にわたり借地借家法に基づく極端な借主保護が続いてきました。これにより、貸主は物価や修繕費、保険料などが上昇しても、自由に家賃を改定することが難しい状況です。また家賃滞納者への対応にも多くの制約があり、市場の健全な運営を妨げています。こうした制度は、戦後の住宅不足を背景とした時代には一定の意義を持っていましたが、現代の実情とは大きく乖離しています。

実際には、周辺相場が上昇していても家賃を見直すには借主の同意または裁判所の判断が必要であり、裁判に持ち込んでも容易には認められません。一方、日用品やサービスは売り手が価格を決定し、買い手が選ぶ市場原理が働いており、住居賃料のみが例外的に硬直化しています。これは資本主義経済の大原則に反しており、不公平な制度といえます。

さらに、家賃滞納が発生しても、契約解除や明渡しまでに半年以上を要することもあり、貸主は長期にわたって損失を被ります。こうした制度が結果的に「居座り型」の脱法行為を助長し、他の善良な借主が間接的にその負担を肩代わりする構造が生まれています。

諸外国と比較しても、日本の制度は特異です。たとえば米国やドイツでは、一定の通知期間を設ければ家賃改定や契約終了が比較的柔軟に行えます。滞納があれば迅速な強制退去が可能な国もあり、日本の「一度貸したら戻らない」制度は世界的に見ても異常に借主側へ偏っています。

このような制度の硬直性を解消し、賃貸市場を現代にふさわしいものとするためには、法制度の見直しとともにAI・データ技術の活用が不可欠です。以下のような施策を提案します。

1. **AIによる「適正家賃」の可視化**
   物件の立地・築年数・設備・周辺相場・更新年数などのビッグデータを活用し、AIが「適正家賃水準」を算出する仕組みを導入することで、貸主が合理的な根拠を持って家賃改定を提案しやすくなります。これにより、借主とのトラブルも軽減できると考えます。

2. **「家賃インデックス制度」の創設**
   国や自治体がAIを用いて地域ごとの家賃インデックス（指数）を定期公表し、物価連動型の家賃調整を可能にする制度を導入することで、双方の合意に頼らずとも市場に沿った家賃の見直しが容易になります。

3. **AI・IoTによる管理業務の効率化と人手不足対応**
   高齢化と人手不足が進む中、賃貸管理業務の省力化は喫緊の課題です。宅配ボックスの稼働状況や満空データをクラウドで可視化し、清掃や荷物回収の最適化を図ります。さらに、スマートロックや遠隔監視システムを導入することで、鍵の管理やトラブル対応の効率が飛躍的に向上します。AIが住民の申告や設備異常の傾向を学習し、事前対応につなげる仕組みも整えます。

制度面では、①家賃改定を物価変動など客観的指標に基づいて一定幅で認めるルール、②家賃滞納が適切な猶予期間（例：2か月）を超えた場合の迅速な契約解除・退去命令制度、③更新時の契約見直し自由度の拡大などが必要であると考えます。これらをAI・データ活用とセットで実施することで、公平で効率的な賃貸市場の実現が可能となります。

旧来型の制度に固執するのではなく、現代のテクノロジーと制度改革を融合させ、すべての関係者が安心・納得して暮らせる住環境を築くべきです。

## 変更提案について

本マニフェストによる変更提案は２つの形で受付をしています。

* [こちら](https://policy.team-mir.ai/view/README.md)のページから、チャットで政策要望を提出する  
* （玄人向け）[GitHub](https://github.com/team-mirai/policy/pulls)でPull Requestを提出する

　頂いた変更提案は、チームみらいが責任をもって確認します。変更提案をマニフェストに反映させるかどうかについては、チームみらいが権限を持ちます。当初マニフェストとは異なる方向性の提案であっても、真剣に検討し、反映するかどうかを考えます。

## 注意事項

* お互いに敬意を持ったコミュニケーションをお願いします。  
* 建設的な議論を心がけてください。  
* わかりやすく簡潔な表現をお願いします。  
* 情報を共有するときは、信頼性のあるデータや引用元に基づいていることを示してください。  
* 注意事項に反して健全な議論が行われていないと判断した場合、一時的に当該スレッドのインタラクションを制限することがあります。
* **AIの活用と責任についての明確な原則**：本プロジェクトではAI（いどばたAI、チャットBot等）を政策議論の支援として活用していますが、AIは意見や文案を生成する道具であり、いかなる責任も負うことはできません。意思決定、説明責任、倫理的判断はすべて人間の側にあるという原則を厳格に守ります。チームみらいは「責任を取れる人が判断する」ことの重要性を再確認し、この原則に基づいて全ての政策運用を行います。

## リンク切れチェックについて

このリポジトリでは、マークダウンファイル内のリンク切れを自動的に検知する仕組みを導入しています。

### 仕組み

- Pull Requestが作成されると、GitHub Actionsによって自動的にマークダウンファイル内のリンクがチェックされます
- 内部リンク（リポジトリ内の他のファイルへのリンク）と外部リンク（ウェブサイトへのリンク）の両方をチェックします
- リンク切れが発見された場合、PRのチェック結果に表示されます

### リンク切れが発見された場合

- PRのチェック結果で報告されたリンクを修正してください
- コミットを追加すると、自動的に再チェックが行われます
